Griffin Menu Studio 1.1.0 — Auto size & a smoother editor

This update makes fitting a menu to one page effortless and irons out several rough edges in the Edit Menu column.

## What's new

- **Auto size.** A new **Auto size** button (in the top bar, and on the overflow warning) sizes the whole menu to one page in either direction — it *grows* the text to fill the page when there's room, and *shrinks* it (spacing first, then type size) when it's too big. It uses the same trusted measurement as Print/Export, so what it says fits, prints.
- **Dismiss the overflow warning.** The "doesn't fit on one page" warning can now be dismissed with the ✕ if you deliberately want an oversized menu — it comes back on its own if the menu changes or later fits.
- **Reorder subtitles and lines freely.** In the Edit Menu column you can now drag a subtitle *below* a line, put a line *above* a subtitle, and interleave them however you like — the printed menu follows the same order.

## Fixes

- **Section options no longer cut off.** On short sections, the ⋯ menu (including **Delete section**) is no longer clipped by the section box.
- The **Auto size** button now sits perfectly centred in the top bar.
- The **"+ Line at top"** control is no longer clipped at the top of the column.
- Removed the redundant **"+ Subtitle at end"** button.
- **Templates** are now saved with a dedicated **.template** extension, keeping the Templates folder tidy and unambiguous.

Thank you for using Griffin Menu Studio.
